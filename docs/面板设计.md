# PrismPanel 面板设计

> 文档状态：第一版正式设计  
> 适用范围：PrismPanel Web 面板、面板后端、节点接入与子服插件协作  
> 设计基线：单 Minecraft 网络、多 Linux 节点、Spigot/Paper 生态  
> 更新日期：2026-07-14

---

## 一、文档定位

本文定义 PrismPanel 第一版正式面板的产品结构、权限模型、通信方式、核心业务、前端页面和交付边界。

当前 `frontend/`、`panel/` 和 `daemon/` 中已经存在的页面与接口主要用于验证守护进程健康状态和实例管理闭环，不作为正式面板的信息架构约束。正式重构时以本文为准。

若本文与现有文档存在冲突，面板相关决策按以下优先级处理：

1. 本文已经明确的面板产品与通信决策。
2. `守护进程设计.md` 中未被本文修改的进程、文件和部署规则。
3. `策划.md` 中未被本文修改的项目原则和后续路线。

本文明确覆盖并修正两个既有分歧：

- 面板主动连接守护进程，守护进程监听管理端口，不采用守护进程主动连接面板的模式。
- 守护进程在第一版以 root 模式运行；Minecraft 实例必须在启动时降权到专用非 root 系统用户。

## 二、产品目标与边界

### 2.1 产品目标

PrismPanel 是面向单个 Minecraft 网络的多节点中控面板，用于统一管理 Linux 节点、代理服、子服、玩家、插件、网络访问规则、性能数据和运维权限。

面板应优先解决以下问题：

- 管理员能够从一个界面判断所有节点和服务器是否健康。
- 不同部门和用户只能查看、操作其被授权的资源。
- 节点、实例、插件和玩家操作具有明确结果、任务进度和审计记录。
- 高风险操作在执行前明确展示影响范围，并阻止无权限或误触操作。
- 插件、配置和 OP 名单由面板集中维护，可安全地批量下发到多个子服。
- 高频日志和文件内容可以凭短期票据直连守护进程，避免面板成为不必要的数据中转瓶颈。

### 2.2 第一版支持范围

| 领域 | 第一版范围 |
|---|---|
| 网络范围 | 一个面板管理一个 Minecraft 网络 |
| 节点 | 多 Linux 节点，面板主动连接守护进程 |
| 服务端 | Spigot、Paper 及兼容 Bukkit/Paper API 的服务端 |
| 代理服 | Velocity、BungeeCord |
| 实例 | 普通实例、镜像服务器组及其派生实例 |
| 用户 | 登录、用户、部门、角色模板、细粒度权限、资源范围 |
| 玩家 | 在线列表、人数、踢出、玩家白名单、聊天消息、跨服转移、全服 OP |
| 插件 | 本地仓库、多版本、配置包、批量部署、状态、启用/禁用、回滚 |
| 节点运维 | 性能统计、节点终端、网络访问白名单 |
| 治理 | 任务、告警、危险操作预警、操作审计 |

### 2.3 第一版明确不做

- 用户自助注册。
- 邮箱、手机号、OAuth 和双因素认证。
- 多租户或多个独立 Minecraft 网络。
- Fabric、Forge、NeoForge 等模组服。
- LuckPerms 权限组管理或同步。
- 玩家封禁与解除封禁。
- 通用插件强制热加载、热卸载。
- 宝塔或其他第三方防火墙面板兼容。
- QQ 机器人和飞书通知的实际发送适配器。
- 多面板实例高可用和跨面板任务调度。
- Docker/cgroup 实例、自动备份与灾难恢复。

QQ 机器人和飞书作为后续通知提供者预留扩展接口，但不成为第一版运行依赖。

## 三、核心对象与关系

### 3.1 术语

| 对象 | 含义 |
|---|---|
| 节点 | 一台运行 Prism 守护进程的 Linux 主机 |
| 服务器组 | 面板中的逻辑服务配置，可以是普通单实例或镜像实例组 |
| 实例 | 节点上实际运行的一个 Java 服务端进程 |
| 代理服 | Velocity 或 BungeeCord 实例，负责网络入口与玩家转移 |
| 子服插件 | 安装在 Spigot/Paper 服务端中的 Prism 插件 |
| 代理插件 | 安装在 Velocity/BungeeCord 中的 Prism 插件 |
| 插件发布 | 一个不可变的插件 JAR、配置目录和元数据版本 |
| 部门 | 用户组，用于批量授予权限；一个用户可以加入多个部门 |
| 任务 | 需要跟踪进度、结果、日志或多个目标的异步操作 |
| 网络白名单 | 将指定节点端口仅开放给指定 IP/CIDR 的防火墙规则 |

### 3.2 对象关系

```text
单个 Minecraft 网络
├── 多个节点
│   ├── 多个服务器组
│   │   └── 一个或多个实例
│   └── 网络访问白名单
├── 一个或多个代理服
├── 全服 OP 名单
├── 插件仓库与发布版本
└── 用户、部门、权限、任务、告警与审计
```

第一版不增加“网络”“项目”或“租户”表。所有节点、服务器和全服 OP 默认属于当前唯一 Minecraft 网络。

## 四、总体架构

### 4.1 组件关系

```text
┌────────────────────────────────────────────────────┐
│ Web 浏览器 / 后续 Wails Windows 客户端             │
│ 登录、页面交互、任务与告警、短期票据直连           │
└──────────────────────┬─────────────────────────────┘
                       │ HTTPS/WSS 或 HTTP/WS
┌──────────────────────▼─────────────────────────────┐
│ PrismPanel 面板后端                                 │
│ 用户权限 · 节点连接 · 配置权威源 · 任务 · 审计      │
│ 插件仓库 · OP 名单 · 指令路由 · 指标聚合            │
└──────────────┬───────────────────┬─────────────────┘
               │ 面板主动连接      │ 本地文件/数据库
          HTTP(S) + WS(S)           ├── MySQL
               │                   └── 插件与配置仓库
┌──────────────▼─────────────────────────────────────┐
│ 节点守护进程，root 模式                             │
│ 进程监督 · 文件 · 控制台 · 指标 · nftables · 终端   │
└──────────────┬───────────────────┬─────────────────┘
               │ 本机认证通道       │ 降权启动
       ┌───────▼────────┐     ┌────▼────────────────┐
       │ Prism 子服插件 │     │ MC/代理服 Java 进程 │
       │ TPS/玩家/插件  │     │ 专用非 root 用户    │
       └────────────────┘     └─────────────────────┘
```

### 4.2 组件职责

| 组件 | 负责 | 不负责 |
|---|---|---|
| 前端 | 登录、权限感知展示、操作确认、状态和日志展示 | 保存节点长期令牌、直接绕过面板执行控制操作 |
| 面板后端 | 用户与权限、节点连接、期望配置、任务、审计、插件仓库、OP 权威数据 | 直接管理 Linux 子进程或防火墙 |
| 守护进程 | 节点本地进程、文件、控制台、性能、终端、防火墙、部署执行 | 用户认证、跨节点权限判断、全局 OP 权威数据 |
| 子服插件 | TPS/MSPT、玩家、OP 校验、Bukkit 插件状态和玩家操作 | Linux 进程、节点文件和防火墙 |
| 代理插件 | 网络玩家视图、聊天路由、跨服转移 | 节点进程和 Bukkit OP |

### 4.3 数据存储

| 数据 | 存储方式 |
|---|---|
| 用户、部门、权限、节点、服务器期望配置 | MySQL |
| 任务、告警、审计、分钟级性能统计 | MySQL |
| 插件 JAR 和配置发布包 | 面板本地受控目录 |
| 插件发布元数据和 SHA-256 | MySQL |
| 节点实际实例配置与运行恢复数据 | 守护进程本地原子 JSON |
| 控制台和部署实时缓冲 | 守护进程有界内存缓冲 |
| 临时直连票据 | 守护进程内存，有界且自动过期 |

第一版不引入 Redis、消息队列和独立时序数据库。

## 五、登录、用户与权限

### 5.1 登录与初始化

- 前端所有业务页面必须登录后访问。
- 不提供注册页面和公开注册接口。
- 首位超级管理员通过面板 CLI 一次性创建，不设置默认账号或默认密码。
- 用户名和密码登录成功后建立服务端 Session。
- Session ID 仅通过 `HttpOnly` Cookie 保存；HTTPS 下必须使用 `Secure`，并设置合适的 `SameSite`。
- 密码使用内存困难型算法哈希，优先选择 Argon2id。
- 登录失败按账号和来源 IP 双维度限流，不泄露账号是否存在。
- 超级管理员可以创建、禁用、删除用户，重置密码并强制全部会话下线。
- 普通用户只能修改自己的密码和退出自己的会话。

### 5.2 用户删除

“删除用户”在产品上表现为删除账户，但数据库保留不可登录的删除标记和不可变用户 ID，以维持审计链完整：

- 删除后立即撤销全部 Session、部门关系、角色和独立授权。
- 审计记录保留当时的用户 ID、用户名快照和操作来源。
- 已删除用户名第一版不允许重新使用。
- 系统必须始终至少保留一个可登录的超级管理员。
- 超级管理员不能删除当前正在使用的最后一个超级管理员账户。

### 5.3 部门与角色

- 部门等价于用户组，一个用户可以属于多个平级部门。
- 第一版不提供部门层级、继承树和显式拒绝规则。
- 权限由角色模板、部门授权和用户独立授权取并集。
- 用户独立授权用于少量例外，不替代部门和角色作为主要管理方式。
- 超级管理员拥有不可裁剪的系统级全部权限。

内置角色模板：

| 角色 | 典型能力 |
|---|---|
| 超级管理员 | 全部能力，用户管理、系统设置和权限授予 |
| 管理员 | 管理授权范围内的节点、服务器、插件和玩家，不管理用户 |
| 运维人员 | 生命周期、控制台、文件、部署和玩家操作 |
| 观察者 | 只读状态、性能、玩家列表和控制台输出 |

### 5.4 资源范围

权限授权必须同时包含动作和资源范围：

```text
全部资源
指定节点
指定服务器组
指定实例
```

下级资源自动受到上级范围约束。例如用户获得某节点的 `server.view` 后，可以查看该节点下的服务器；获得某服务器组的 `instance.restart` 后，不能重启同节点其他服务器。

### 5.5 主要权限码

| 领域 | 权限示例 |
|---|---|
| 节点 | `node.view`、`node.create`、`node.update`、`node.delete` |
| 终端 | `node.terminal`、`node.terminal.root` |
| 服务器 | `server.view`、`server.create`、`server.configure`、`server.delete` |
| 生命周期 | `instance.start`、`instance.stop`、`instance.restart`、`instance.kill` |
| 控制台 | `console.read`、`console.command` |
| 文件 | `file.read`、`file.write`、`file.delete` |
| 玩家 | `player.view`、`player.kick`、`player.message`、`player.transfer` |
| 玩家白名单 | `player.whitelist.manage` |
| OP | `player.op.manage` |
| 插件 | `plugin.view`、`plugin.upload`、`plugin.deploy`、`plugin.remove` |
| 网络白名单 | `firewall.view`、`firewall.manage` |
| 任务 | `task.view`、`task.cancel`、`task.retry` |
| 告警与审计 | `alert.view`、`alert.acknowledge`、`audit.view` |
| 用户系统 | `user.manage`、`department.manage`、`permission.manage` |
| 系统 | `system.settings` |

`user.manage`、`department.manage`、`permission.manage` 和向其他用户授予 root 终端权限，只允许超级管理员持有。

## 六、节点接入与连接管理

### 6.1 添加节点流程

```text
在 Linux 节点启动守护进程
-> 守护进程首次启动生成长期随机节点令牌
-> 管理员在面板点击“新增节点”
-> 填写节点名称、连接 URL、令牌和可选公网 URL
-> 面板连接守护进程并完成认证、版本和能力检查
-> 保存节点并维持管理 WebSocket 长连接
-> 断线后按退避策略自动重连
```

连接地址使用完整 URL，支持 IP 或域名：

```text
http://192.168.1.20:24444
http://203.0.113.20:24444
https://node.example.com:24444
```

节点令牌由守护进程生成，可通过本地命令查看和重置。重置后现有面板连接失效，需要更新面板中的节点凭据。

### 6.2 认证协议

正式协议不直接在 WebSocket 中发送长期节点令牌，使用挑战响应：

1. 面板发送客户端随机数、面板协议版本和时间戳。
2. 守护进程返回服务端随机数、节点 ID 和能力声明。
3. 面板使用节点令牌对双方随机数、节点 ID 和时间戳计算 HMAC。
4. 守护进程校验时间窗口、随机数唯一性和 HMAC。
5. 认证成功后建立控制会话，双方协商协议版本。

挑战响应降低长期令牌在公网 HTTP 中直接泄露的风险，但不提供传输加密。

### 6.3 HTTP 与 HTTPS

- 第一版允许面板通过公网 HTTP/WS 连接守护进程。
- HTTPS/WSS 仍是推荐方式。
- 公网 HTTP 节点在节点列表、节点详情和危险操作确认中持续显示“连接未加密”。
- HTTP 下的命令、日志、文件内容和性能数据可能被监听或篡改，面板不得把挑战响应描述成等价于 TLS。
- 节点配置需要记录连接安全级别，供审计和告警使用。

### 6.4 节点状态

```text
CONNECTING   正在连接或重连
ONLINE       管理连接正常
DEGRADED     已连接但版本、插件或部分能力异常
OFFLINE      连接中断
DISABLED     管理员禁用，不再自动连接
```

面板不得把节点离线误报为节点上的 Minecraft 实例已经停止。离线只表示无法获得当前实际状态。

### 6.5 root 模式与实例降权

守护进程第一版以 root 运行，用于：

- 管理节点防火墙。
- 提供授权后的 root 节点终端。
- 创建和维护不同实例的目录与所有权。
- 将 Java 进程降权到实例专用用户后启动。

每个服务器组必须配置 `run_as_user`，并满足：

- 不允许使用 `root`、UID 0 或无法解析的系统用户。
- 启动 Java 进程前设置目标 UID、GID、补充组和工作目录。
- 工作目录、日志和部署文件归对应实例用户所有。
- 启动命令即使使用 Shell，也必须在完成降权后执行。
- 守护进程不得因为自身是 root 而让普通实例继承 root 权限。

### 6.6 节点终端

- 节点终端默认仅超级管理员可用。
- 超级管理员可以单独向其他用户授予 `node.terminal` 或 `node.terminal.root`。
- 每次打开终端创建限时 PTY 会话，绑定用户、节点、来源 IP 和会话 ID。
- 会话记录输入、输出、开始时间、结束时间和终止原因，用于审计回放。
- 面板支持超级管理员强制结束其他终端会话。
- root 终端属于严重风险操作，进入前需要再次确认目标节点。

## 七、服务器、实例与玩家

### 7.1 服务端平台

第一版平台类型：

```text
PAPER       Paper 及声明兼容的分支
SPIGOT      Spigot/Bukkit API 服务端
VELOCITY    Velocity 代理服
BUNGEE      BungeeCord 代理服
```

服务端平台决定可使用的插件类型、指标来源和玩家操作能力。没有安装 Prism 子服插件或代理插件时，面板仍可管理进程和控制台，但游戏内数据标记为不可用。

### 7.2 服务器组与实例

- 普通服务器组对应一个固定实例。
- 镜像服务器组维护一个镜像目录和多个派生实例，可批量部署。
- 服务器组记录所属节点、平台、显示名称、启动参数、停止命令、端口、实例用户和自动重启策略。
- 实例记录实际工作目录、运行端口、状态、PID、启动时间和插件连接状态。
- 第一版不做跨节点自动调度和实例自动迁移。

### 7.3 生命周期

```text
STOPPED -> STARTING -> RUNNING -> STOPPING -> STOPPED
                         │
                         └───────────────> FAILED
```

- 同一实例的生命周期操作必须串行执行。
- 正常停止先发送停止命令，超时后终止进程组，最后强杀。
- 面板执行停服或重启前显示当前在线玩家数量。
- 强杀必须输入实例名称确认。
- 节点离线时生命周期操作立即失败，不在恢复连接后偷偷执行旧操作。

### 7.4 子服插件通信

Prism 子服插件与代理插件只连接本机守护进程：

```text
子服/代理插件 -> 127.0.0.1 或 Unix Socket -> 守护进程 -> 面板
```

- 守护进程启动实例时为其生成实例级短期身份材料。
- 插件认证必须绑定实例 ID、启动代次和本机连接来源。
- 插件不得直接监听公网端口或持有面板节点令牌。
- 实例每次重启后身份代次变化，旧连接自动失效。

### 7.5 玩家管理

子服插件上报：

- 玩家 UUID、名称、当前子服、连接时间和延迟。
- 子服在线人数和最大人数。
- 玩家加入、离开和跨服事件。

第一版玩家操作：

| 操作 | 执行组件 | 说明 |
|---|---|---|
| 查看玩家 | 子服/代理插件 | 支持按子服筛选 |
| 踢出玩家 | 玩家所在服务端或代理插件 | 必须填写原因 |
| 发送消息 | 子服/代理插件 | 明确标记为管理系统消息，不冒充玩家 |
| 转移玩家 | Velocity/Bungee 代理插件 | 校验目标服在线和玩家仍在线 |
| 玩家白名单 | 子服插件 | 与网络访问白名单分开 |
| OP 管理 | 子服插件 | 仅接受面板权威名单 |

聊天消息支持单个玩家、指定子服、多个子服和全网络。第一版不提供定时广播。

### 7.6 全服 OP

- 面板数据库是全服 OP 的唯一权威源。
- OP 以玩家 UUID 为主键，同时保存最近玩家名。
- OP 名单应用到所有 Spigot/Paper 子服，代理服不参与。
- 面板控制台必须拒绝 `op` 和 `deop` 命令。
- Prism 子服插件必须拦截玩家和服务端控制台执行的 `/op`、`/deop`。
- 其他插件直接修改 OP 状态时，Prism 插件检测配置偏差、恢复面板期望状态并上报告警。
- OP 增删只能通过具有 `player.op.manage` 权限的面板用户执行，并写入审计日志。

## 八、插件仓库与部署

### 8.1 插件发布模型

每个插件可以有多个不可变发布版本：

```text
插件标识
显示名称
适用平台
版本号
JAR 文件
基础配置目录
SHA-256
上传者与上传时间
更新说明
兼容的 Minecraft/服务端版本
```

- JAR 和配置文件保存在面板本地受控目录。
- 数据库只保存元数据、相对存储路径和文件摘要。
- 上传完成后先写临时文件，校验成功再原子发布。
- 已被部署记录引用的发布版本不可原地覆盖；修改内容必须创建新版本。
- 第一版不实现配置变量模板。不同配置组合通过不同发布版本表达。

### 8.2 部署目标

插件发布可以部署到：

- 单个实例。
- 一个服务器组的全部实例。
- 多个选中的服务器组或实例。

部署前检查平台、服务端版本、目标目录、磁盘空间、实例状态和现有插件版本。

### 8.3 部署方式

- 安装、升级、降级、启用、禁用和移除均创建任务。
- 批量部署支持先选一个实例灰度执行，再继续其余目标。
- 每个目标独立记录成功或失败，任务不得只返回模糊的整体结果。
- 部署前保存被替换的 JAR 和受管理配置，用于版本回滚。
- 目标正在运行且操作需要重启时，界面必须显示影响玩家数并要求确认。

### 8.4 启用、禁用与重载

- 不提供通用强制热加载和热卸载。
- 启用或禁用插件修改期望部署状态，在安全重启后生效。
- 对明确声明支持重载的插件，可以配置受约束的重载命令。
- 重载命令仍通过面板权限、危险提示和审计，不允许插件发布携带任意 root Shell 命令。

### 8.5 运行状态

Prism 子服插件上报每个 Bukkit 插件的：

- 名称、版本、主类和启用状态。
- 加载失败和禁用原因。
- 与面板期望版本是否一致。
- 是否存在 JAR 或配置漂移。

面板分别展示“期望状态”和“实际状态”，不得把 JAR 已复制等同于插件已经正常运行。

## 九、网络访问白名单

### 9.1 产品语义

网络访问白名单用于把节点上的指定端口开放给指定 IP 或 CIDR。规则支持：

- 单端口，例如 `25565`。
- 端口范围，例如 `25565-25575`。
- 端口列表，例如 `25565,25570,25580-25590`。
- TCP、UDP 或两者。
- IPv4 地址、IPv4 CIDR、IPv6 地址和 IPv6 CIDR。
- 备注、启用状态和资源权限范围。

对于被 PrismPanel 管理的端口，匹配白名单的来源放行，未匹配来源拒绝。未被管理的端口不受影响。

### 9.2 nftables 边界

- 守护进程以 root 直接执行 nftables 原子事务，不增加独立特权服务。
- 只维护 `table inet prismpanel` 及其内部 chain/set。
- 不清空、不重排、不修改其他工具或管理员创建的规则。
- 所有变更先生成完整候选规则，语法检查通过后原子应用。
- 应用失败时保留上一版本，面板任务返回明确错误。
- 守护进程启动和重连时读取实际规则并与面板记录对账。
- 面板外部修改 PrismPanel 专用表时，记录配置漂移告警，不静默覆盖。

### 9.3 风险控制

- 新规则必须通过端口、协议、地址和重复冲突校验。
- 修改或删除规则属于严重风险操作，需要展示节点、端口和受影响来源。
- 面板管理连接端口默认禁止被网络白名单功能接管，避免远程锁死节点。
- 防火墙任务记录变更前后规则摘要，并支持恢复上一成功版本。

## 十一、性能统计与告警

### 11.1 指标来源

守护进程上报节点和操作系统进程指标，子服插件上报游戏内指标。

节点指标：

- CPU 使用率、系统负载。
- 内存使用量和使用率。
- 各受关注磁盘容量和使用率。
- 网络收发速率。
- 守护进程版本、运行时长和连接延迟。

实例指标：

- CPU、RSS、运行时长和重启次数。
- JVM 堆、GC 次数/耗时和线程数。
- TPS、MSPT、在线人数和玩家延迟。
- Prism 插件连接状态和数据更新时间。

### 11.2 采样与保留

- 守护进程每 5 秒上报实时指标。
- 面板实时界面使用最新样本，不把每个高频样本永久写入数据库。
- 面板按 1 分钟聚合并保存 30 天。
- 第一版使用 MySQL 保存分钟级数据，不引入时序数据库。
- 数据过期由定时清理任务分批删除，避免大事务阻塞。

### 11.3 告警

第一版站内告警包括：

- 节点离线或连接长期不安全。
- 实例启动失败、异常退出或频繁重启。
- TPS/MSPT 异常。
- CPU、内存或磁盘超过阈值。
- 插件连接丢失、版本漂移或加载失败。
- 插件部署、防火墙或镜像部署失败。
- OP 状态漂移或出现被拒绝的 OP 命令。

告警支持未处理、已确认、已恢复状态。顶部栏告警入口展示未处理数量。

通知模块使用提供者接口设计，预留：

```text
QQBotProvider
FeishuProvider
```

第一版不实现提供者配置和实际消息发送。

## 十二、危险操作与审计

### 12.1 危险等级

| 等级 | 示例 | 前端要求 |
|---|---|---|
| 普通 | 查看、启动、发送普通管理消息 | 常规确认或直接执行 |
| 高风险 | 停服、重启、插件部署、文件覆盖 | 二次确认，展示玩家和目标数量 |
| 严重 | 强杀、删除、root 终端、防火墙、OP、权限变更 | 输入目标名称或明确确认语句 |

前端提示不能替代后端权限校验。危险等级、确认要求和权限必须由后端返回或再次校验，不能只写在前端代码中。

### 12.2 审计内容

所有状态变更操作记录：

- 操作 ID、时间、用户 ID、用户名快照和部门快照。
- 来源 IP、Session ID 和客户端信息。
- 动作、资源类型、资源 ID 和显示名称快照。
- 请求摘要、变更前后差异和危险等级。
- 成功、失败、错误码、任务 ID 和节点请求 ID。
- 对节点 HTTP/HTTPS 安全级别的记录。

控制台命令记录完整命令。密码、节点令牌、临时票据和插件私密配置必须脱敏。

节点终端单独记录会话元数据和输入输出回放。终端审计文件使用受限目录存储，并在数据库中保存摘要和索引。

审计记录不随用户、节点或服务器删除而删除。第一版提供按用户、动作、资源、结果和时间筛选。

## 十三、票据直连

### 13.1 允许直连的能力

| 范围 | 用途 |
|---|---|
| `console.read` | 只读控制台日志 |
| `deployment.read` | 只读部署任务日志 |
| `file.list` | 文件树和目录列表 |
| `file.read` | 文件读取 |
| `file.upload` | 文件上传 |
| `file.download` | 文件下载 |
| `file.edit` | 指定文件编辑保存 |

启停、重启、强杀、控制台命令、部署执行、文件删除、网络白名单和权限变更始终经过面板。

文件上传和编辑虽然由浏览器直连执行，但必须先由面板鉴权、登记待执行操作并在完成后回报审计结果。

### 13.2 票据约束

票据至少绑定：

- 随机票据 ID 和不可预测密钥。
- 用户、节点、实例或任务。
- 单一权限范围。
- 指定路径、HTTP 方法和文件大小上限。
- 签发时间、过期时间和使用次数。
- 面板审计操作 ID。

文件请求使用 `Authorization: Bearer`；WebSocket 在首条消息中发送票据，不把票据放进 URL。

公网 HTTP 下票据仍可能被监听，因此默认使用短有效期和单次使用。HTTP 不保证文件或日志内容的机密性。

### 13.3 代理回退

- 守护进程配置 `public_url` 时，面板可以向前端返回直连地址。
- `public_url` 为空、浏览器不可达或跨域/证书检查失败时，前端通过面板代理。
- 代理和直连必须使用相同权限语义、错误码和审计链。

## 十四、正式前端设计

### 14.1 视觉基线

正式前端参考 `docs/前端设计.html` 的深色侧栏、浅色工作区、状态色和服务器卡片，但不直接沿用其占位页面和大圆角营销式卡片。

- 使用 Vue 3、Vite 和 Element Plus。
- 图标使用项目本地依赖，不依赖 CDN。
- 运维界面强调扫描效率，主要卡片圆角控制在 6-8px。
- 表格、筛选器和任务状态保持高信息密度。
- 桌面端优先，同时支持平板和手机。
- 手机端侧栏改为抽屉，不保留固定窄栏。

### 14.2 一级导航

侧栏保持单层标签，不按“资源/运维/治理”分组：

```text
总览
服务器
插件
用户
网络白名单
节点
```

导航项根据权限隐藏。隐藏只改善体验，后端仍必须独立鉴权。

顶部栏包含：

- 当前页面标题和面包屑。
- 运行中任务入口和数量。
- 未处理告警入口和数量。
- 当前用户、修改密码和退出登录。

### 14.3 登录页

- 只有用户名、密码和登录按钮。
- 不显示注册、找回密码、第三方登录和功能营销内容。
- 首次未创建超级管理员时，只显示需要通过 CLI 初始化的状态，不允许网页初始化。
- 登录错误使用统一提示，不说明用户名是否存在。

### 14.4 总览页

总览与服务器列表完全分开，不提供全局节点下拉框。

顶部显示紧凑的全局摘要：

```text
在线玩家 | 在线节点 | 运行实例 | 当前告警
```

下方按节点竖向排列全宽节点卡片，一个节点一张卡片：

```text
┌─────────────────────────────────────────────────────────┐
│ 节点名称                    在线状态 · 延迟 · HTTP 风险  │
│ CPU  内存  磁盘  网络收发                               │
│ 运行实例/总实例  在线玩家  最低 TPS  活跃任务  告警数   │
└─────────────────────────────────────────────────────────┘
```

- 性能使用进度条、数字和小型趋势线，不在卡片内部嵌套更多卡片。
- 告警节点优先显示，离线节点保留最后数据并标注更新时间。
- 点击节点卡片进入节点详情。
- 无权查看某节点的用户不看到该节点及其汇总数据。

### 14.5 服务器页

服务器页提供独立列表，支持：

- 按节点、平台、状态和服务器组筛选。
- 名称、ID 和端口搜索。
- 网格/表格视图切换。
- 显示状态、平台、节点、在线人数、TPS、插件连接和告警。
- 按权限提供新建、批量启动、停止、重启和部署入口。

服务器详情内部标签：

```text
概览 | 控制台 | 玩家 | 性能 | 插件 | 文件 | 配置 | 操作记录
```

全服 OP 和玩家白名单入口放在服务器页的玩家管理区域，不新增一级导航。

### 14.6 插件页

插件页是全局插件仓库，主要区域包括：

- 插件列表和平台、版本筛选。
- 上传插件和创建发布版本。
- 发布详情、文件摘要、配置目录和更新说明。
- 已部署目标、实际运行状态和版本漂移。
- 批量部署、灰度、回滚、启用和禁用任务。

插件页不出现“通用热加载/热卸载”按钮。

### 14.7 用户页

用户页内部标签：

```text
用户 | 部门 | 角色与权限 | 操作日志
```

- 用户列表展示状态、部门、角色、最后登录和活跃会话。
- 创建用户仅超级管理员可见。
- 权限编辑器先选择角色/部门，再设置资源范围和少量用户例外。
- 操作日志支持筛选和查看任务、命令、终端会话关联。

### 14.8 网络白名单页

- 页面只表示 Linux 网络访问规则，不混入 Minecraft 玩家白名单。
- 列表展示节点、协议、端口表达式、IP/CIDR、状态、备注和最后应用结果。
- 创建/编辑使用结构化端口输入，不接受任意 nftables 文本。
- 应用变更通过任务抽屉展示结果。

### 14.9 节点页

节点列表展示：

- 名称、地址、连接状态、延迟和协议版本。
- HTTP/HTTPS 安全级别。
- CPU、内存、磁盘、实例数和告警数。
- 守护进程版本、最后连接时间和能力状态。

节点详情内部标签：

```text
概览 | 性能 | 实例 | 网络白名单 | 终端 | 连接与设置 | 操作记录
```

新增节点对话框填写名称、连接 URL、节点令牌和可选 `public_url`，保存前必须执行连接测试。

### 14.10 任务与告警入口

任务和告警不占用一级导航：

- 点击顶部任务图标打开任务抽屉。
- 点击顶部告警图标打开告警抽屉。
- 抽屉中的条目可以进入独立详情路由。
- 运行中任务和严重告警在移动端仍必须可访问。

| `nodes` | 名称、连接 URL、加密令牌、状态和版本 |
| `server_groups` | 平台、节点、进程、端口和期望配置 |
| `instances` | 派生实例、实际状态和插件连接状态 |
| `operators` | 全服 OP UUID、名称和变更信息 |
| `player_whitelist` | Minecraft 玩家白名单 |
| `plugins` / `plugin_releases` | 插件元数据、发布版本、文件摘要 |
| `plugin_deployments` | 目标期望版本和实际结果 |
| `firewall_rules` | 节点、协议、端口、来源和应用状态 |
| `tasks` / `task_targets` | 任务主体和目标级结果 |
| `metrics_1m` | 节点与实例分钟级聚合指标 |
| `alerts` | 告警类型、资源、状态和时间 |
| `audit_logs` | 不可变操作审计记录 |

节点长期令牌必须使用面板主密钥加密存储，不能以明文写入数据库或日志。

## 十七、失败处理与一致性

- 面板保存期望配置，守护进程报告实际配置和修订号。
- 节点断线后保留最后状态和时间戳，不伪造实时状态。
- 控制类操作在节点离线时快速失败；不会在重连后延迟执行过期命令。
- 配置发布和批量部署使用任务，可明确重试。
- 同一实例的生命周期、部署和插件变更必须互斥。
- 所有跨网络请求带请求 ID，写操作带幂等键。
- 面板超时后先查询节点结果，再决定是否重试。
- 插件、OP 和防火墙同时保存期望状态和实际状态，配置漂移必须可见。
- 删除节点、服务器、插件发布或用户时，不级联删除审计和历史任务。

## 十八、实施顺序

第一版作为一个产品版本交付，但按以下顺序实现和验收：

### 阶段 A：控制面基础

- MySQL 数据模型和迁移。
- 登录、Session、超级管理员 CLI。
- 用户、部门、角色和资源权限。
- 多节点连接管理与挑战响应认证。
- 正式前端壳、总览、节点和服务器页面。
- 统一错误、请求 ID 和审计框架。

### 阶段 B：服务器与游戏数据

- 服务器组、实例和生命周期重构。
- Prism 子服插件与代理插件通信。
- TPS/MSPT、玩家列表、聊天和跨服转移。
- 玩家白名单和全服 OP 权威同步。
- 控制台、文件和票据直连。

### 阶段 C：集中运维

- 统一任务中心。
- 插件仓库、发布、批量部署和回滚。
- 网络访问白名单和 nftables 管理。
- 节点终端与回放审计。
- 分钟级性能统计、站内告警和总览完善。

每个阶段必须形成可运行闭环，不在阶段 A 中预制尚未使用的消息队列、时序数据库或多租户抽象。

## 十九、第一版验收标准

1. 未登录用户无法访问任何业务数据或业务 WebSocket，系统不存在注册入口。
2. 超级管理员能够创建、删除和禁用用户；删除用户后审计记录仍可追溯。
3. 部门、角色、用户例外授权和节点/服务器资源范围能够正确组合。
4. 面板能够主动连接多个守护进程，断线重连不会丢失其他节点连接。
5. 公网 HTTP 节点可以工作，同时在界面和审计中被明确标记为未加密。
6. 守护进程以 root 运行时，Minecraft Java 进程仍以配置的非 root 用户运行。
7. 总览按节点竖向展示全宽性能卡片，离线和告警节点状态清晰可见。
8. 服务器列表与总览分离，并能按节点、平台、状态筛选。
9. Prism 插件能够上报 TPS、MSPT、玩家和插件运行状态。
10. 玩家消息和跨服转移具有明确成功/失败结果并写入审计。
11. `/op` 和 `/deop` 在玩家、控制台和面板命令入口均被拒绝；OP 只能由面板修改。
12. 插件发布不可原地覆盖，批量部署提供目标级结果、灰度和回滚。
13. 通用插件热加载/卸载不可用，启用/禁用遵循重启生效规则。
14. 网络白名单支持单端口、端口范围、端口列表和 IP/CIDR，并只修改 PrismPanel 自己的 nftables 表。
15. 高频控制台、部署日志和文件传输可以凭有界短期票据直连，权限过期后立即失效。
16. 停服、重启、插件部署、强杀、root 终端、防火墙和权限变更符合危险分级确认规则。
17. 节点、实例、JVM 和游戏指标能够实时展示，并保留 30 天分钟级历史。
18. 任务、告警、控制台命令、终端、插件、防火墙、用户和 OP 操作均可审计。
